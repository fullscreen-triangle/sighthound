# Validation experiments — Buffer-Processor Model of Rail Network Yield

Theorem-by-theorem numerical validation of the paper
`../german-rail-network-yield.tex`. Each experiment isolates one claim and
checks it against an independently computed ground truth (brute-force
enumeration on a small network, where the global optimum is exactly computable).

## Files

- `model.py` — the model: section network, geography, block-second quantum,
  vehicles with convex energy cost, items with private targets, yield functional,
  separation cost, deterministic closure, clearing prices. Kept small-scale so
  every check has exact ground truth. Depends on `numpy` only.
- `run_experiments.py` — the five experiments and the JSON runner.
- `results.json` — output (regenerated by running the script).

## Run

```
python run_experiments.py
```

## Experiments and what each checks

| # | Theorem | Check | Ground truth |
|---|---------|-------|--------------|
| 1 | Occupancy–Closure–Clearing Equivalence (Thm 6.4) | yield-optimal ⇔ closed ⇔ cleared, and clearing price = separation cost per used section | brute-force optimum over task-delivering configs; all 58 delivering single-section neighbours are strictly dominated |
| 2 | Forced optimal speed + interior optimum (Thm 7.3, Def 7.1) | closed config runs at v\*; v\* → ceiling as energy weight λ→0, interior optimum when energy binds; each v\* is a local max | fine speed-grid sweep of the yield ratio |
| 3 | Comparative-advantage sorting (Thm 7.6) | on the Nuremberg–Berlin (VDE 8) corridor, the high-capability vehicle strictly wins the 300 km/h sections; on speed-limited sections neither has an advantage | per-section yield density D(u,σ) |
| 4 | Liveness (Thm 8.6) | greedy target-ward routing drains every buffer in finite rounds; residual distance monotone non-increasing per item | discrete-round simulation on a corridor with a feeder branch |
| 5 | Quantum coincidence (Prop 6.5) | splitting β₀ into distinct physical/algorithmic/economic constants degrades the equivalence exactly as predicted; unifying them recovers exact coincidence and zero price band | analytic prediction vs measured gaps |

## Notes on the model choices

- **Yield is a ratio** (useful displacement / block-second cost). A pure
  unconstrained ratio optimum degenerately picks a single high-yield
  block-second, so Experiments 1 and 5 optimize over configurations that
  actually **deliver the transport task** (all items reach their targets) — the
  meaningful optimum for a fixed demand.
- **Gradient-connectedness** (Def 2.2, universal all-pairs) does not hold on a
  strictly one-way corridor. Liveness (Thm 8.6) only needs the weaker
  **target-relative** condition — every item can descend toward *its own* target
  — which `model.Network.target_descendable` checks and Experiment 4 verifies.
- **β₀ in yield units**: the resolution floor bounds *resolvable* yield
  differences. We take one block-second's share of the optimum's yield
  (`y_star / total-block-seconds`) as the yield-unit floor; closure and clearing
  are checked against this threshold, per the "up to β₀" qualifier in Thm 6.4.

All five experiments currently pass (`all_pass: true` in `results.json`).
